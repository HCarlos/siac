Resumen del módulo

El módulo CentroBot permite a sistemas externos (bots de WhatsApp, SMS u otros canales) generar un enlace único para que el ciudadano suba fotos de su solicitud de servicio municipal, sin necesidad de crear una cuenta ni iniciar sesión.

Flujo de 3 pasos:
1. El bot llama a GET /centrobot/generar-url-fotos?solicitud_id=123 y obtiene una URL única.
2. El bot envía esa URL al ciudadano (WhatsApp, SMS, etc.).
3. El ciudadano abre la URL en su navegador y sube sus fotos desde la página web.
Público objetivo de esta documentación: desarrolladores que integren bots externos con SIAC, técnicos del municipio y operadores que quieran entender cómo funciona la subida pública de imágenes.

Endpoint 1 — Generar URL

GET /centrobot/generar-url-fotos Consumida por el bot externo

Genera una URL pública única (basada en el UUID de la solicitud) para que el ciudadano suba fotos. No requiere autenticación.

Parámetros de consulta (query string)

ParámetroTipoReq.Descripción
solicitud_id integer Sí ID (folio) de la solicitud. Debe ser entero mayor a 0.

Ejemplo de petición

GET /centrobot/generar-url-fotos?solicitud_id=4521 HTTP/1.1
Host: siac.villahermosa.gob.mx

Respuesta exitosa  200 OK

{
  "status":  true,
  "mensaje": "URL generada correctamente.",
  "url":     "https://siac.villahermosa.gob.mx/a3f7c2d1-84b0-4e5a-9f12-0123456789ab"
}

Respuestas de error

HTTPCondiciónRespuesta
422 solicitud_id faltante, no entero o ≤ 0 Respuesta estándar de validación Laravel
200 Folio no existe en la base de datos { "status": false, "mensaje": "No existe ese folio..." }
Protección SQL: solicitud_id se valida como integer|min:1 antes de llegar a la base de datos. Eloquent usa PDO con prepared statements — inyección SQL imposible.

Endpoint 2 — Página de Subida

GET /{uuid} Página web para el ciudadano

Muestra la página pública donde el ciudadano puede ver los detalles de su solicitud y subir fotos. No requiere autenticación.

Parámetro de ruta

ParámetroTipoDescripción
uuid string — UUID v4 Identificador único de la solicitud. Formato: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

Ejemplo de petición

GET /a3f7c2d1-84b0-4e5a-9f12-0123456789ab HTTP/1.1
Host: siac.villahermosa.gob.mx

Respuesta

Página HTML completa (Bootstrap 5) con:

  • Detalles de la solicitud (folio, servicio, dirección, fecha, descripción)
  • Zona de carga con soporte para drag & drop y selección múltiple
  • Galería de fotos ya subidas con vista previa
Validación de UUID en ruta: la expresión regular [a-f0-9]{8}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{4}-[a-f0-9]{12} garantiza que solo se acepten UUIDs v4 válidos. Cualquier otra cadena devuelve 404.

Endpoint 3 — Subir Foto

POST /fotos/subir/{uuid} Una llamada por imagen

Recibe y guarda una imagen enviada por el ciudadano. El frontend llama a este endpoint una vez por cada imagen seleccionada.

Content-Type: multipart/form-data

Parámetro de ruta

ParámetroTipoDescripción
uuid string — UUID v4 UUID de la solicitud (mismo que en la URL pública)

Campos del formulario (form-data)

CampoTipoReq.Restricciones
foto file Sí JPG, JPEG, PNG, GIF o WEBP únicamente.
Máximo 10 MB por imagen.
Sin extensiones peligrosas en el nombre.
Sin código PHP / script embebido.
observaciones string No Texto descriptivo. Máximo 500 caracteres.
momento string No Momento de la fotografía. Valores permitidos: ANTES o DESPUÉS.
Si se omite, el valor predeterminado es ANTES.
_token string Sí Token CSRF de Laravel (el frontend lo lee del meta tag de la página).

Ejemplo de petición (cURL)

curl -X POST \
  "https://siac.villahermosa.gob.mx/fotos/subir/a3f7c2d1-84b0-4e5a-9f12-0123456789ab" \
  -F "foto=@/ruta/a/mi_foto.jpg" \
  -F "observaciones=Bache en la esquina norte" \
  -F "momento=ANTES" \
  -F "_token=TOKEN_CSRF_AQUI"

Respuesta exitosa  200 OK

{
  "status":        true,
  "mensaje":       "¡Foto subida correctamente!",
  "imagen_id":     1042,
  "url_imagen":    "https://siac.villahermosa.gob.mx/storage/denuncia/abc123...jpg",
  "url_thumb":     "https://siac.villahermosa.gob.mx/storage/denuncia/_thumb_abc123...png",
  "observaciones": "Bache en la esquina norte",
  "momento":       "ANTES"
}

Seguridad — 4 Capas de Validación

El endpoint POST /fotos/subir/{uuid} aplica 4 capas de validación independientes en cascada. Si cualquiera falla, el archivo es rechazado y no se escribe nada en disco.

1
Validación Laravel — MIME + getimagesize() + tamaño
Reglas image + mimes:jpg,jpeg,png,gif,webp + max:10240. La regla image usa getimagesize() sobre el contenido real del archivo — no solo la extensión. SVG está explícitamente excluido porque puede contener JavaScript embebido (XSS).
Laravel Validator getimagesize() Sin SVG
2
Magic Bytes — Firma binaria con PHP finfo
Se lee la firma binaria real con finfo_file(FILEINFO_MIME_TYPE) y se compara contra los MIME permitidos (image/jpeg, image/png, image/gif, image/webp). Detecta archivos cuya extensión fue cambiada manualmente (p.ej. un ejecutable renombrado a .jpg).
PHP finfo FILEINFO_MIME_TYPE
3
Doble Extensión — Nombre original del archivo
Se analiza cada segmento del nombre original separado por puntos. Si alguno coincide con una extensión peligrosa, el archivo es rechazado. Protege contra ataques tipo malware.php.jpg o script.js.png. Extensiones bloqueadas: php, phar, exe, sh, asp, js, html, svg, xml…
Doble extensión Nombre original
4
Escaneo de Cabecera — Código malicioso embebido
Se leen los primeros 2 KB del archivo y se buscan patrones de código: <?php, eval(, exec(, system(, <script, base64_decode(, etc. Previene imágenes JPEG/PNG con PHP insertado en los metadatos EXIF.
EXIF injection 2 KB scan

Otras protecciones activas

ProtecciónDetalle
Nombre en disco Generado con sha1(timestamp + time()) — el nombre original nunca se usa para almacenamiento.
CSRF El POST requiere token CSRF de Laravel. El GET no lo necesita (solo lectura).
UUID en ruta Regex estricta en el router — solo acepta UUIDs v4 válidos, rechaza cualquier otra cadena.
solicitud_id Validado como integer|min:1; Eloquent usa PDO prepared statements (no hay inyección SQL).
latitud / longitud Se toman de $denuncia->latitud y $denuncia->longitud — no se aceptan coordenadas del cliente.
momento Validado con in:ANTES,DESPUÉS; cualquier otro valor es reemplazado por ANTES como fallback.
Thumbnails regenerados Las miniaturas se generan con Intervention Image, destruyendo cualquier payload en metadatos del original.

Tipos de Imagen Permitidos

Extensión(es)MIME TypeNotas
.jpg   .jpegimage/jpegFormato más común para fotografías
.pngimage/pngSoporta transparencia

Ejemplos cURL

1 · Generar URL para la solicitud #4521

curl "https://siac.villahermosa.gob.mx/centrobot/generar-url-fotos?solicitud_id=4521"

2 · Abrir la página de subida (en el navegador)

# Simplemente pegar esta URL en cualquier navegador:
https://siac.villahermosa.gob.mx/a3f7c2d1-84b0-4e5a-9f12-0123456789ab

3 · Subir una foto con cURL

# El token CSRF se obtiene desde la página web cargada en el navegador.
# En el flujo normal el navegador lo envía automáticamente.

curl -X POST \
  "https://siac.villahermosa.gob.mx/fotos/subir/a3f7c2d1-84b0-4e5a-9f12-0123456789ab" \
  -H "X-CSRF-TOKEN: TU_TOKEN_CSRF_AQUI" \
  -F "foto=@/home/usuario/foto_bache.jpg" \
  -F "observaciones=Bache profundo en Av. Méndez esquina con Calle 5" \
  -F "momento=ANTES"
